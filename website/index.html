Containers to triples

A project to bring light on the interior of the Docker Images.

What do we do?

Due to the variety of operative systems, frameworks and applications we have on these days, it´s becoming more and more complicated to make an app work in the same way in every system. Differences in the functionality or incompatibilities are some of the main problems that can appear due to this variety.

Docker is a tool that helps us to solve this problem. With Docker you can package your app with all the dependencies and needs in a container that everyone can download in their own computer and launch the app from inside this container. This way we ensure that this app is going to work the same way in all different computers and systems.

But not everything is perfect. These Docker containers can look like a black box as it´s really difficult to know what exactly is inside. ¿Which operative system are we using? ¿Which different tools are we using to run the app? These are some of the questions Containers to Triples helps you to answer.

INSPECT

We inspect the inside of your container to extract all the main metadata and most important information.

DEPENDENCIES

We look for all the different dependencies that our container is using.

KNOWLEDGE GRAPH

We build a knowledge graph with all this information so everyone can make simple consults and get all the info they need.
